# 재난안전 AI 시범 서비스 — 화면설계서 (1차 POC)

- 작성일: 2026-07-23 · 버전 v0.2 (착수 사업수행계획서 v0.9의 시범서비스 모듈 5종 반영)
- 디자인 시스템: UNE Design System v1.0.0 (`UNE Design System/une-design-system/project/`)
- 글꼴: SpoqaHanSansNeo (Regular/Medium/Bold, 로컬 자체 호스팅 — `apps/web/public/fonts/`)
- 상위 근거: docs/00_사업수행계획서_분석.md 4절 — ①재난상황 입력 패널 ②지도 통합 뷰어 ③유사사례 Top-K 목록 ④상세정보 조회 ⑤정합성(Verification)

## 0. 디자인 원칙 (UNE DS 준수사항)

- **색**: 화이트 서피스 `#FFFFFF` + 잉크 텍스트 `#1C202A`, 프라이머리 `#3C69FC`(light-blue-500). 시맨틱: 에러 `#D92D20` / 경고 `#E89A00` / 성공 `#1D792B`. 다크 모드는 토큰 스코프(`:root[data-theme="dark"]`)로 지원.
- **타이포**: Spoqa Han Sans Neo 전용, letter-spacing -3%. `typo-title-lg|md|sm`(32/24/20), `typo-body-lg|md|sm`(16/14/12) 클래스 사용.
- **아이콘**: UNE 250종(`assets/icons/icon-data.js`, 한국어 키, currentColor)만 사용. 신규 아이콘 드로잉·이모지 금지.
- **UI 문구**: 한국어, 짧은 명사구(확인, 취소, 더보기, 알림). 마케팅 톤·느낌표 금지.
- **로고**: 브랜드 마크 없음 — "une"를 플레인 타입으로 표기.
- **모서리/그림자**: 카드·모달 radius md(8)~lg(12), 입력 sm(4), 배지 pill(1000). 그림자는 2레이어(ambient+key) 토큰.

## 2. 영역별 상세

### 2.1 헤더 (Header 컴포넌트)

|---|---|---|

### 2.2 ① 상황입력 패널 (SituationPanel)
재난메타 사건(EVT) 골격에 대응하는 입력 폼.

| 필드 | 컴포넌트 | 값 | 메타 매핑 |
|---|---|---|---|
| 재난유형 | ChoiceChip 5개 | 호우/홍수/수해·침수/태풍/산사태 (사업 5종 — HEAVY_RAIN·FLOOD·INUNDATION·TYPHOON·LANDSLIDE, T코드 병기) | EVT-08 주위해 |
| 경보 종류 | Segment | 기상특보 / 위기경보 | NAT-02 특보·경보체계 |
| 경보 단계 | Segment | (특보) 주의보·경보 / (위기경보) 관심·주의·경계·심각 / (산사태) 주의보·예비경보·경보 — 유형에 따라 동적 | EVT-10 사건규모등급 |
| 발생 일시 | Datepicker | 기본값 현재 | EVT-04 발생일시 |
| 피해/통제 키워드 | InputChip (자유 추가) | 예: 침수·지하차도·하수 역류·차량고립·반지하 (사업수행계획서 예시 키워드 프리셋 제공) | 질의 구조화 keywords |
| 상황 메모 | Textarea | 자유 기술 (예: "시간당 50mm 강우 예상") | 사건 설명 |
| 적용 | Button (primary) | 상황 확정 → event_id 발급(형식: 유형-일시-행정코드-순번)·헤더 배지·검색 필터 갱신 | — |

- "적용" 시 입력 전체가 구조화 질의 JSON(Query Builder)으로 변환됨 — 누락 없는 수집이 검증 체크리스트 1단계.
- 유형 선택 시 판단기준 탭이 해당 유형 기준으로 자동 전환.
- 지자체·유형 조합에 자료가 없으면(예: 남원+저감계획) Nonmodal로 "해당 지자체는 하천기본계획 자료만 제공됩니다" 안내.

### 2.3 ② 지도 통합 뷰어 (MapViewer) — L1~L4 공간표현 전략
| 요소 | 구현 | 동작 |
|---|---|---|
| 베이스맵 | Leaflet + OSM 타일 | 선택 지자체로 초기 이동·줌. 다크 테마 시 다크 타일 |
| L1 마커 | 위험지구 대표점·시설 | 클릭 → ④상세조회 카드 열기. 재해유형별 색(상태·색상 매핑 규칙) |
| L2 폴리곤/라인 | 하천 구간·침수 면 | 하천 선택 시 구간 하이라이트, 측점 팝업(계획홍수량·홍수위) |

| 연동 | Top-K 카드 ↔ 마커 | 카드 선택 시 지도 이동·하이라이트(동일 사건 기준) |

### 2.4 질의·응답 + ⑤ 정합성 (AskPanel, 지도 하단)
| 요소 | 컴포넌트 | 동작 |
|---|---|---|
| 질의 입력 | Input + IconButton(검색) | Enter 전송, 전송 중 disabled. 예시 질의 ActionChip 3개(빈 상태) |
| 응답 카드 | Card + Badge | 근거 기반 응답 본문(typo-body-md) + 재난유형 Badge + rag_answer_id(툴팁) |
| ⑤ Citation | ListItem ×n | 「문서명 · 장 · p.xx-yy」 + 인용문 1줄. 클릭 시 전문 Modal. **모든 응답에 필수(근거 없는 답변 금지)** |
| ⑤ 요인별 기여도 | 수평 바 (토큰 색) | 유형·공간·피해·문맥 유사도 기여(similarity_factors) — "왜 이 결과인지" 표시 |
| ⑤ 검증 정보 토글 | Switch | 검색 점수 상세 등 내부 검증 정보 표시(공개 화면에서는 기본 숨김) |
| 구조화 참조 | ActionChip | 응답 연계 위험지구/하천 → 우측 탭 해당 항목 + 지도 하이라이트 |
| 대기 상태 | Spinner | 응답 생성 중 |

- 모든 응답 하단에 고지 문구(typo-body-sm, text-disabled): "본 응답은 계획 문서 기반 참고 정보입니다."

### 2.5 우측 탭 — ③ 유사사례 Top-K 목록 / ④ 상세조회 / 판단기준

**탭 A. 유사사례 (Top-K Results)** — /api/ask·/api/search 결과
- 결과 카드 리스트 (Card): 제목(문서·지구/사례명) + 재해유형 Badge + **유사도 점수**(Badge, %) + 요약 2줄 + 출처
- 정렬(유사도/최신)·FilterChip(문서유형·재해유형), 카드 클릭 → 상세 팝업(Modal) + 지도 연동
- 빈 상태: Empty + "재난 상황을 적용하고 질의를 실행하십시오"

**탭 B. 상세조회 (Case Detail)** — districts.json·rivers.json

*위험지구* (선택 지자체)
- 요약 스탯: 위험지구 수(유형별 분포), 총사업비 — 예: 구미 66개소·4,216억원 / 의왕 17개소·1,074억원
- 필터: FilterChip (재해유형 8종: 하천/내수/사면/토사/바람/가뭄/대설/기타)
- 지구 카드 (Card): 지구명 + 재해유형 Badge + 위험등급 DotBadge(초고위험~저위험) + 위치 + 예상피해액 + 투자우선순위
- 카드 클릭 → 상세 Modal: 위험요인·위험조건(임계값) / 저감대책(공종·규모) / 사업비·재원·사업상태 / 연차별 시행계획 / 근거(관리대장 p.x)

*하천* (선택 지자체)
- 하천 개요 카드: 하천명 · 등급(국가/지방) Badge · 유역면적 · 연장 · 시점/종점
- 계획홍수량 표 (Table): 지점(측점) | 계획홍수량(m³/s) | 계획홍수위(EL.m) | 특보 기준유량(50%/70%) | 비고
- 홍수 판단 연계 안내 카드: "홍수주의보=계획홍수량 50% 유량 수위, 경보=70%" (criteria 연계)
- 자료 없는 지자체(의왕 1차): Empty + "안양천 하천기본계획은 원본(HWP) 변환 후 제공 예정"

**탭 C. 판단기준** (선택된 재난유형 기준 — criteria.json)
- 판단조건 카드: 판단주체(기상청 등) Badge + 정량 기준표(Table: 단계 | 기준 | 관련 변수)
  - 예: 호우 — 주의보 3h 60/12h 110mm · 경보 3h 90/12h 180mm · 극한호우 1h 50&3h 90
- 등급 스테퍼: 단계(주의보→경보 / 관심→주의→경계→심각)를 DotBadge+색상으로 표기, 현재 입력 단계 하이라이트
- 게이지(상황 메모에 수치 입력 시): 입력값/기준값 비율 바 — 예: "3h 강우 75mm / 경보 기준 90mm (83%)"
- 근거 출처: ListItem (S# · 기관 · URL)

## 3. 상태·색상 매핑 규칙

| 도메인 값 | UNE 토큰/색 |
|---|---|
| 위기경보 관심(Blue) | light-blue-500 `#3C69FC` |
| 위기경보 주의(Yellow) / 특보 주의보 | yellow `#E89A00` |
| 위기경보 경계(Orange) | orange 스케일 (fig-tokens orange-500) |
| 위기경보 심각(Red) / 특보 경보 / 초고위험 | red `#D92D20` |
| 저위험/정상 | green `#1D792B` |
| 위험도 등급(저→초고) | green → yellow → orange → red 4단 |

- 색상만으로 의미 전달 금지 — 항상 텍스트 라벨 병기 (고대비 모드 `[data-mode="high-contrast"]` 지원).

## 4. 컴포넌트 조달 방식
- `apps/web/src/ds/`에 UNE DS의 `styles.css`·`tokens/`·필요 컴포넌트(JSX)·`assets/icons/`를 복사해 사용 (원본 폴더는 읽기 전용 유지).
- 폰트: `tokens/fonts.css`의 CDN 로드를 제거하고 로컬 `@font-face`(public/fonts/SpoqaHanSansNeo-*.ttf)로 교체 — 지침 3(제공 글꼴 사용) 및 오프라인 데모 대비.
- 사용 예정 컴포넌트: Header, Select, ChoiceChip, FilterChip, ActionChip, InputChip, Segment, Switch, Datepicker, Textarea, Input, Button, IconButton, Card, Badge, DotBadge, ListItem, Table(HeaderCell/BodyCell/Slot), Tab, Modal, Toast, Spinner, Empty, Nonmodal, Tooltip.
- 지도: Leaflet(오픈소스)를 사용하되 컨트롤·팝업의 색·타이포는 UNE 토큰으로 오버라이드(지도 내부도 DS 룩 유지). 마커·폴리곤 색은 3장 상태·색상 매핑 규칙을 따름.

## 5. 접근성·모드
- 키보드: 질의 입력 Enter 전송, 탭 이동 focus-visible 링(`--color-focus-ring`) 유지.
- 다크/고대비/모바일 토큰 스코프는 DS 기본 제공 — 1차 POC는 라이트+다크 2모드 검증까지.
