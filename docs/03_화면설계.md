# 재난안전 AI 시범 서비스 — 화면설계서 (1차 POC)

- 작성일: 2026-07-23 · 버전 v0.1
- 디자인 시스템: UNE Design System v1.0.0 (`UNE Design System/une-design-system/project/`)
- 글꼴: SpoqaHanSansNeo (Regular/Medium/Bold, 로컬 자체 호스팅 — `apps/web/public/fonts/`)

## 0. 디자인 원칙 (UNE DS 준수사항)

- **색**: 화이트 서피스 `#FFFFFF` + 잉크 텍스트 `#1C202A`, 프라이머리 `#3C69FC`(light-blue-500). 시맨틱: 에러 `#D92D20` / 경고 `#E89A00` / 성공 `#1D792B`. 다크 모드는 토큰 스코프(`:root[data-theme="dark"]`)로 지원.
- **타이포**: Spoqa Han Sans Neo 전용, letter-spacing -3%. `typo-title-lg|md|sm`(32/24/20), `typo-body-lg|md|sm`(16/14/12) 클래스 사용.
- **아이콘**: UNE 250종(`assets/icons/icon-data.js`, 한국어 키, currentColor)만 사용. 신규 아이콘 드로잉·이모지 금지.
- **UI 문구**: 한국어, 짧은 명사구(확인, 취소, 더보기, 알림). 마케팅 톤·느낌표 금지.
- **로고**: 브랜드 마크 없음 — "une"를 플레인 타입으로 표기.
- **모서리/그림자**: 카드·모달 radius md(8)~lg(12), 입력 sm(4), 배지 pill(1000). 그림자는 2레이어(ambient+key) 토큰.

## 2. 영역별 상세

### 2.1 헤더 (Header 컴포넌트)
| 요소 | 컴포넌트 | 내용 |
|---|---|---|
| 서비스명 | 플레인 타입 | "une 재난안전 AI" (typo-title-sm) |
| 지자체 선택 | Select (MD 40) | 의왕시 / 구미시 / 남원시 — 선택 시 전 패널 필터 갱신 |
| 위기경보 표시 | Badge | 현재 입력된 상황의 위기경보 단계. 관심=파랑, 주의=노랑(#E89A00), 경계=주황 계열, 심각=빨강(#D92D20) |

### 2.2 ① 상황입력 패널 (SituationPanel)
재난메타 사건(EVT) 골격에 대응하는 입력 폼.

| 필드 | 컴포넌트 | 값 | 메타 매핑 |
|---|---|---|---|
| 재난유형 | ChoiceChip 5개 | 태풍/풍수해/호우/홍수/산사태 | EVT-08 주위해(T코드) |
| 경보 종류 | Segment | 기상특보 / 위기경보 | NAT-02 특보·경보체계 |
| 경보 단계 | Segment | (특보) 주의보·경보 / (위기경보) 관심·주의·경계·심각 / (산사태) 주의보·예비경보·경보 — 유형에 따라 동적 | EVT-10 사건규모등급 |
| 발생 일시 | Datepicker | 기본값 현재 | EVT-04 발생일시 |
| 상황 메모 | Textarea | 자유 기술 (예: "3시간 90mm 예상") | 사건 설명 |
| 적용 | Button (primary) | 상황 확정 → 헤더 배지·검색 필터 갱신 | — |

- 유형 선택 시 판단기준 탭이 해당 유형 기준으로 자동 전환.
- 지자체·유형 조합에 자료가 없으면(예: 남원+저감계획) Nonmodal로 "해당 지자체는 하천기본계획 자료만 제공됩니다" 안내.

### 2.3 ② AI 질의응답 (ChatPanel)
| 요소 | 컴포넌트 | 동작 |
|---|---|---|
| 대화 스레드 | Card 리스트 | 사용자 질의(우측 정렬 말풍선) / AI 응답(Card) |
| AI 응답 카드 | Card + Badge | 응답 본문(typo-body-md) + 상단에 재난유형 Badge |
| 근거 인용 | ListItem ×n | 「문서명 · 장 · p.xx-yy」 + 인용문 1줄. 클릭 시 인용문 전문 Modal |
| 구조화 참조 | ActionChip | 응답에 연계된 위험지구/하천 → 클릭 시 ③ 패널 해당 탭·항목으로 이동 |
| 대기 상태 | Spinner | 응답 생성 중 |
| 빈 상태 | Empty + 안내 | "재난 상황을 입력하고 질의를 시작하십시오" + 예시 질의 ActionChip 3개 |
| 입력 | Input + IconButton(검색) | Enter 전송, 전송 중 disabled |

- 모든 응답 하단에 고지 문구(typo-body-sm, text-disabled): "본 응답은 계획 문서 기반 참고 정보입니다."

### 2.4 ③ 시각화 패널 (VizPanel) — Tab 3종

**탭 A. 판단기준** (선택된 재난유형 기준 — criteria.json)
- 판단조건 카드: 판단주체(기상청 등) Badge + 정량 기준표(Table: 단계 | 기준 | 관련 변수)
  - 예: 호우 — 주의보 3h 60/12h 110mm · 경보 3h 90/12h 180mm · 극한호우 1h 50&3h 90
- 등급 스테퍼: 단계(주의보→경보 / 관심→주의→경계→심각)를 DotBadge+색상으로 표기, 현재 입력 단계 하이라이트
- 게이지(상황 메모에 수치 입력 시): 입력값/기준값 비율 바 — 예: "3h 강우 75mm / 경보 기준 90mm (83%)"
- 근거 출처: ListItem (S# · 기관 · URL)

**탭 B. 위험지구** (선택 지자체 — districts.json)
- 요약 스탯: 위험지구 수(유형별 분포), 총사업비 — 예: 구미 66개소·4,216억원 / 의왕 17개소·1,074억원
- 필터: FilterChip (재해유형 8종: 하천/내수/사면/토사/바람/가뭄/대설/기타)
- 지구 카드 리스트 (Card): 지구명 + 재해유형 Badge + 위험등급 DotBadge(초고위험~저위험) + 위치 + 예상피해액 + 투자우선순위
- 카드 클릭 → 상세 Modal: 위험요인 / 저감대책(공종·규모) / 사업비·재원 / 연차별 시행계획 / 근거(관리대장 p.x)

**탭 C. 하천** (선택 지자체 — rivers.json)
- 하천 개요 카드: 하천명 · 등급(국가/지방) Badge · 유역면적 · 연장 · 시점/종점
- 계획홍수량 표 (Table): 지점(측점) | 계획홍수량(m³/s) | 계획홍수위(EL.m) | 비고
- 홍수 판단 연계 정보: "홍수주의보=계획홍수량 50% 유량 수위, 경보=70%" 안내 카드 (criteria 연계)
- 자료 없는 지자체(의왕 1차): Empty + "안양천 하천기본계획은 원본(HWP) 변환 후 제공 예정"

## 3. 상태·색상 매핑 규칙

| 도메인 값 | UNE 토큰/색 |
|---|---|
| 위기경보 관심(Blue) | light-blue-500 `#3C69FC` |
| 위기경보 주의(Yellow) / 특보 주의보 | yellow `#E89A00` |
| 위기경보 경계(Orange) | orange 스케일 (fig-tokens orange-500) |
| 위기경보 심각(Red) / 특보 경보 / 초고위험 | red `#D92D20` |
| 저위험/정상 | green `#1D792B` |
| 위험도 등급(저→초고) | green → yellow → orange → red 4단 |

- 색상만으로 의미 전달 금지 — 항상 텍스트 라벨 병기 (고대비 모드 `[data-mode="high-contrast"]` 지원).

## 4. 컴포넌트 조달 방식
- `apps/web/src/ds/`에 UNE DS의 `styles.css`·`tokens/`·필요 컴포넌트(JSX)·`assets/icons/`를 복사해 사용 (원본 폴더는 읽기 전용 유지).
- 폰트: `tokens/fonts.css`의 CDN 로드를 제거하고 로컬 `@font-face`(public/fonts/SpoqaHanSansNeo-*.ttf)로 교체 — 지침 3(제공 글꼴 사용) 및 오프라인 데모 대비.
- 사용 예정 컴포넌트: Header, Select, ChoiceChip, FilterChip, ActionChip, Segment, Datepicker, Textarea, Input, Button, IconButton, Card, Badge, DotBadge, ListItem, Table(HeaderCell/BodyCell/Slot), Tab, Modal, Toast, Spinner, Empty, Nonmodal, Tooltip.

## 5. 접근성·모드
- 키보드: 질의 입력 Enter 전송, 탭 이동 focus-visible 링(`--color-focus-ring`) 유지.
- 다크/고대비/모바일 토큰 스코프는 DS 기본 제공 — 1차 POC는 라이트+다크 2모드 검증까지.
