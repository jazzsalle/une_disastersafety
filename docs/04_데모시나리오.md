```bash
# 리포 루트에서 실행 — localhost:8000 기존 서버 사용, 없으면 uvicorn 자체 기동·종료
```

  체크리스트 ①~⑧을 자동 판정해 표로 출력한다. 종료 코드: 전부 통과 0 / 실패 1.
- **외부 호출 차단**: 자체 기동 모드에서는 자식 프로세스 환경에서
  `ANTHROPIC_API_KEY`·`UNI_RAG_ACCOUNT`·`UNI_RAG_PASSWORD`·`UNI_RAG_MODEL_KEY`를 제거해
  mock 경로로 고정한다(anthropic·UNI RAG 서버 호출 없음). 기존 서버 사용 모드에서는
  서버 기동 시 환경을 따른다.
- 체크리스트 ⑧(로그)은 자체 기동 모드에서 서버 stdout 캡처 로그의 `rag_answer_id=` 라인으로

## 2. 시나리오 S1 — 호우/의왕 (T10107 · 41430)

### 목적
호우 특보(경보) 상황에서 의왕시 침수 위험지구(하천·내수재해)와 대응 사례를
근거(Citation)와 함께 조회하고, 지도 L1 위험지구 표출·목록 연동을 확인한다.

### 상황입력 패널 입력값
| 항목 | 값 |
|---|---|
| 지자체(헤더) | 경기도 의왕시(41430) — 기본 선택값 |
| 재난유형 | 호우 T10107 |
| 경보 종류 / 단계 | 특보 / 경보 |
| 발생 일시 | 기본값(현재) 유지 |
| 피해·통제 키워드 | `침수`, `지하차도` |
| 상황 메모 | `시간당 50mm 강우 예상` (선택) |

### 프론트 클릭 절차
1. 헤더 지자체 드롭다운에서 「경기도 의왕시」 확인(기본값).
2. 좌측 상황입력 패널에서 재난유형 Chip 「호우 T10107」 클릭 → 우측 패널이 판단기준 탭으로 자동 전환(호우 기준 표시).
3. 경보 종류 「특보」·경보 단계 「경보」 선택.
4. 키워드 입력란에 `침수` 입력 후 Enter, `지하차도` 입력 후 Enter(InputChip 2개 생성).
5. [적용] 클릭 → 상단 Toast 「상황 적용 완료 · HEAVY_RAIN-…-41430-…」, 헤더에 경보 Badge(경보=적색) 표시.
6. 하단 질의응답 패널 「근거 응답」 탭에서 질의 `호우 시 침수 위험지구와 대응 사례` 입력 후 Enter.
7. 우측 「유사사례 Top-K」 탭에서 결과 카드 클릭 → 지도 해당 지구 하이라이트·이동 확인.
8. 응답 하단 Citation 항목 클릭 → 출처(문서명·장·페이지·발췌) 모달 확인.

### 예상 결과
- 지도(모듈②): 의왕시 경계(L2)·안양천(L3)과 위험지구 L1 마커 17개소 표출, 카드 클릭 시 하이라이트.
- 상세조회(모듈④)·structured_refs: 위험지구 17개소 — **안양천지구**(하천재해)·**고천A지구**(내수재해) 포함
  (근거: 제2차 의왕시 자연재해저감 종합계획 관리대장 2022.7, `data/manual/districts.json`).
- 정합성(모듈⑤): citations 1건 이상 — 전건 《의왕시 자연재해저감종합계획(2022.7)》 등 문서명·장·페이지 표기,
  요인 기여도 4종(type·time·space·damage) 바 표출(키워드 매칭으로 damage > 0).
- 판단기준 탭: 호우(T10107) 특보 기준(3시간·12시간 강우량) 표.

### API 레벨 검증
`python scripts/verify_scenarios.py` — S1 어서션 17건
(HTTP 200, rag_answer_id, Top-K·요인 4키, citations 전건 출처 필드, geo_refs L1 17건,
districts 안양천·고천 포함, 전건 admin_code=41430 정합 등). 결과: **17/17 PASS**.

mock으로 폴백한다(설계: `docs/06_UNI_RAG_챗봇_연동.md`).

### 절차
1. 하단 질의응답 패널에서 「챗봇 대화」 탭 클릭.
2. 메시지 입력(예: `의왕시 침수 이력 알려줘`) 후 전송 — 현재 적용된 event 컨텍스트가 함께 전달된다.
3. 응답 확인:
   - **UNI RAG 실연동 시**: SSE 스트리밍으로 답변이 점진 표시(mock 배지 없음).
   - **mock 폴백 시**: 완성 응답 1건 + **「mock 응답」 Badge** + 안내 문구
     「UNI RAG 미연결 mock 응답」 + 로컬 코퍼스 발췌(문서명·장·페이지) 표기.
4. 후속 질문 전송 — 직전 대화(history)가 함께 전달되는 연속 대화 확인.

### mock 폴백 확인 방법 (UNI RAG 서버 미가동 시)
- `.env`(또는 셸 환경)에서 `UNI_RAG_ACCOUNT`를 비우고 백엔드 기동 → 챗봇 응답에
  「mock 응답」 Badge·notice가 표시되면 정상(외부 호출 자체가 발생하지 않음).
- 자격증명이 설정된 상태라도 UNI RAG 서버 오류(timeout·4xx·5xx) 시 동일하게 mock 폴백되며,
  서버 로그에 `UNI RAG 연동 실패 — mock 폴백` warning이 남는다(자격증명·토큰은 로그 미기록).
- API 직접 확인: `POST /api/chat` 응답이 JSON(`mode:"mock"`)이면 폴백, `text/event-stream`이면 실연동.

### UNI RAG 실연동 상태
2026-07-23 오후 기준 **모델 서버 가동 — 실연동 동작 확인**(로그인 게이트 개인 JWT 경유).
모델이 영어 추론(thinking)을 본문으로 스트리밍하는 문제는 프론트 SSE 파서가
`</think>` 마커 기준으로 걸러 **최종 한국어 답변만 표시**한다(추론 구간에는 로딩 스피너 유지).
서버 미가동·오류 시 기존대로 mock 폴백.

## 8. AI Agent 인터랙션 시연 시나리오 — 지역별 3종 (발주처 시연용)

지도(모듈②)·챗봇(모듈⑥)이 **AI Agent와 양방향으로 상호작용**하는 흐름을 지역별 1개씩 시연한다.
세 시나리오를 이어서 진행하면 인터랙션 6종(POI 컨텍스트 주입 · 근거 발췌 표기 · 후속 질문 제안 ·
응답 속 지명→지도 연동 · 자연어 지도 제어 · thinking 제거)이 전부 시연된다.

> 사전 상태: 배포 URL 접속 → UNE 계정 로그인 완료. 챗봇 응답에 「mock 응답」 배지가 없으면
> UNI RAG 실연동 상태다(배지가 보이면 모델 서버 미가동 — 시연은 동일 절차로 진행 가능).

### A1. 의왕 — 지도 POI 선택 → 속성 자동 주입 질의 (호우/침수 상황)

**시연 포인트**: "지도에서 지구를 클릭하기만 하면, AI가 그 지구의 속성정보(위치·재해유형·위험요인)를
알고 답합니다" — POI 컨텍스트 주입 + 근거 발췌 + 후속 질문 칩.

| 단계 | 조작 | 확인 포인트 |
|---|---|---|
| 1 | 헤더 지자체 「경기도 의왕시」(기본값) 확인, S1 상황 적용(§2 절차 1~5 — 호우 T10107·특보 경보·키워드 `침수`) | 헤더 경보 Badge(적색) |
| 2 | 지도에서 **안양천지구** L1 마커(파란색 — 하천재해) 클릭 | 팝업 + 우측 상세조회 자동 전환, **챗봇 입력창 위 POI 칩** 「안양천지구 · 하천재해」 표시 |
| 3 | 「챗봇 대화」 탭 클릭 — POI 칩 아래 **후속 질문 제안 칩 3종** 확인 | 「이 지구가 왜 위험지구로 선정되었는지 알려줘」 등 |
| 4 | 제안 칩 1번 클릭(입력 없이 질의) | 추론 과정 없이 **한국어 답변만 스트리밍** — 안양천지구의 실제 위험요인(제방여유고·하폭 부족, 교량 시설능력 부족) 기반 답변 |
| 5 | 답변 하단 **로컬 근거 발췌**(문서명·장·페이지) 확인 | 《의왕시 자연재해저감종합계획》 출처 표기 — XAI 원칙 |
| 6 | POI 칩의 ×로 선택 해제 | 칩 제거 — 이후 질의는 일반 질의로 전송 |

멘트 예시: *"방금 질의에는 '이 지구'라고만 했지만, AI Agent는 지도에서 선택된 안양천지구의
관리대장 속성(위치, 재해유형, 위험요인)을 자동으로 전달받아 그 지구에 대한 답을 정확히 합니다."*

**기대효과 — 지자체 담당자 관점**

| 담당자가 확인할 수 있는 것 | 업무 의의 |
|---|---|
| 호우 특보 발령 직후, 관내 위험지구별 **선정 사유·위험요인**(제방여유고 부족, 교량 시설능력 부족 등)을 지도에서 클릭 즉시 확인 | 1,200여 페이지 저감종합계획·관리대장을 뒤지지 않고 **초동 대응 단계에서 수 초 내 확인** — 상황 판단 시간 단축 |
| 답변마다 붙는 출처(문서명·장·페이지) | 상황보고서·부단체장 보고에 **그대로 인용 가능한 근거** 확보 — "AI가 그랬다"가 아니라 "저감계획 제3장 p.12에 따르면"으로 보고 |
| 후속 질문 제안 칩 | 순환보직 등으로 **재난 업무가 처음인 담당자도** 무엇을 물어야 할지 시스템이 안내 — 담당자 역량 편차 보완 |

### A2. 남원 — 자연어 지도 제어 + 하천 질의 + 응답 지명 클릭→지도 (홍수 상황)

**시연 포인트**: "말로 지도를 움직이고, AI 답변 속 지명을 클릭하면 지도가 따라갑니다" —
자연어 명령 + 하천 POI 질의 + 응답 속 지명 링크.

| 단계 | 조작 | 확인 포인트 |
|---|---|---|
| 1 | (의왕 화면 상태에서) 챗봇에 **`남원으로 이동해줘`** 입력·전송 | LLM 호출 없이 즉시 「전라북도 남원시로 지도를 이동했습니다.」 응답 + 지도·데이터가 남원으로 전환 |
| 2 | 챗봇에 **`요천 보여줘`** 입력·전송 | 「요천 위치로 지도를 이동하고 상세조회를 열었습니다.」 — 지도 요천 하이라이트 + 우측 상세조회(하천 표) |
| 3 | POI 칩 「요천 · 국가하천」 상태에서 제안 칩 **「이 하천의 계획홍수량을 알려줘」** 클릭 | Y0~Y5 산정지점·계획홍수량(예: Y4 남원수위표 2,005㎥/s) 기반 답변 + 근거 발췌 |
| 4 | 답변 본문에 **파란 점선 밑줄 지명**(예: 요천, 남원 지구명) 클릭 | 지도 하이라이트·상세조회 이동 — 대화→지도 역방향 연동 |
| 5 | 후속 질의 `특보 기준유량은?` 직접 입력 | history 연속 대화 — 주의보 1,003 / 경보 1,404㎥/s(계획홍수량 50%/70%) 맥락 답변 |

멘트 예시: *"운영자는 마우스 없이도 '남원으로 이동' 한 마디로 지도를 제어하고,
AI 답변에 나온 지구명을 클릭하면 바로 현장 위치와 상세정보로 이동합니다."*

**기대효과 — 지자체 담당자 관점**

| 담당자가 확인할 수 있는 것 | 업무 의의 |
|---|---|
| 하천 수위 상승 시 산정지점별 **계획홍수량·특보 기준유량**(주의보 50%·경보 70%)을 대화로 즉시 확인 | 하천기본계획 책자 없이 **특보 발령 판단의 참고 수치를 실시간 확보** — 발령 지연 리스크 감소 |
| 답변 속 지명 클릭 → 지도·상세 즉시 이동 | 상황실 브리핑 중 **말→화면 전환이 끊기지 않음** — 현장 출동·교통통제 지점 결정이 빨라짐 |
| "남원으로 이동" 등 자연어 지도 제어 | 긴박한 상황에서 메뉴 조작 숙련도와 무관하게 **누구나 즉시 시스템 운용** — 야간·휴일 당직자도 동일 품질 대응 |

### A3. 구미 — 상황(Event)+POI 결합 질의 → 근거·정합성 확인 (산사태 상황)

**시연 포인트**: "재난 상황 정보와 지구 정보가 함께 AI에 전달되어, 상황 맞춤형 답을 근거와 함께 냅니다."

| 단계 | 조작 | 확인 포인트 |
|---|---|---|
| 1 | 챗봇에 `구미로 이동해줘` 전송(또는 헤더에서 선택) | 지도 구미 전환 |
| 2 | S3 상황 적용(§4 절차 — 산사태 T10401·특보 **주의보**·키워드 `사면`·`급경사지`) | 판단기준 탭에 토양함수지수 80/90/100% 표 |
| 3 | 지도에서 **황산지구**(사면재해 — 노란색 마커) 클릭 | POI 칩 「황산지구 · 사면재해」 |
| 4 | 챗봇 질의 `현재 주의보 상황에서 이 지구에 어떤 대비가 필요한지 알려줘` | 질의에 **상황(산사태·주의보·키워드) + 황산지구 속성 + 로컬 근거 발췌**가 모두 자동 첨부되어 상황 맞춤 답변 |
| 5 | 답변 하단 근거 발췌의 문서·페이지 확인 → 우측 「유사사례 Top-K」 탭에서 동일 근거 카드 대조 | 챗봇·근거응답이 같은 코퍼스(XAI) 기반임을 시연 |
| 6 | (마무리) 헤더 로그아웃 클릭 | 로그인 게이트 복귀 — 개인 계정 기반 접근 제어 시연 |

멘트 예시: *"AI Agent에는 화면에서 설정한 재난 상황과 선택 지구, 그리고 저감종합계획 원문 발췌가
함께 전달됩니다. 향후 T3Q AI Agent로 교체되어도 이 계약(어댑터)은 그대로 유지됩니다."*

**기대효과 — 지자체 담당자 관점**

| 담당자가 확인할 수 있는 것 | 업무 의의 |
|---|---|
| 현재 발령 단계(주의보)에 맞는 **지구별 대비사항** — 같은 질문이라도 상황(단계·키워드)에 따라 달라지는 맞춤 답변 | 단계별 행동요령을 문서에서 찾아 조합하는 대신 **"지금, 이 지구"에 필요한 조치**를 바로 확인 — 예방 점검 누락 방지 |
| 챗봇 근거 발췌와 유사사례 Top-K가 **같은 원문 코퍼스**를 가리킴 | 대화형 답변도 검증 가능한 공식 문서 기반임을 확인 — **AI 답변의 행정 신뢰성** 확보(XAI) |
| 개인 계정 로그인·로그아웃 | 누가 언제 시스템을 사용했는지 계정 단위 관리 — **행정망 보안·책임소재** 요건 충족 |

### 종합 기대효과 — 시연 마무리 멘트용

**"잠자던 계획 문서를 상황실의 실시간 도구로"** — 본 POC가 보여주는 서비스의 본질적 가치:

1. **문서 → 대화형 지식**: 지자체마다 수립해 둔 자연재해저감종합계획·하천기본계획
   (본 POC 기준 문서 6종 **5,745페이지 → 3,974개 근거 단위**)은 평시 서가에 있고 재난 시
   펼쳐볼 시간이 없다. AI Agent는 이 문서를 **상황 발생 순간 지도 위에서 대화로 꺼내 쓰는
   실시간 지식**으로 전환한다.
2. **근거 기반 행정(XAI)**: 모든 답변에 문서명·장·페이지가 따라붙어, AI 활용이
   **감사·보고에 견디는 행정 행위**가 된다 — 사업수행계획서의 근거 제시 원칙 구현.
3. **담당자 역량 표준화**: 순환보직·야간 당직 등 담당자 숙련도 편차와 무관하게,
   지도 클릭·자연어 대화만으로 동일 수준의 상황 정보에 접근한다.
4. **확장성**: 데이터(재난메타 체계)·AI(어댑터 계약)·화면(모듈 5종+챗봇)이 분리되어 있어,
   본사업 대상(부산·인제·영천) 확대와 T3Q RAG/AI Agent 교체 시에도
   **지자체별 문서만 파이프라인에 투입하면 동일 서비스가 성립**한다.
