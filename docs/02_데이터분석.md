## 1. 재난메타 정의서 v0.6 (T3Q 설계 중 자료)

NIA 「재난안전 AI데이터 구축」(RFP 2026.3), TTAK.KO-10.1126/1047 기반. 부처별 분절 데이터를 **개별 사건 단위로 결합·관리**하기 위한 층위별 메타데이터 체계.

### 1.1 구조 요약
- **6개 패싯**: ①재난안전유형 T(L1대→L2중→L3소 3단, 유일한 3단 깊이) ②재난관리단계 P(예방·대비·대응·복구) ③피해대상 S(인적·물적) ④주관기관 A ⑤재난관리자원 R ⑥정보유형 I(정형·반정형·비정형)
- **L0 공통(재난안전정보 레코드)** 5갈래: 정보자원(레코드 33+DOI) · 공간(좌표계·geometry 5) · 지역 AreaUnit(자기참조 트리 — 전체영역⊃시군구⊃법정동⊃세부지역, ARU-01~08) · **사건 Event(EVT-01~12: 사건ID·사건명·발생/종료일시·행정구역·주위해T·연관위해·규모등급·선포상태·상위사건)** · AI(후속)
- 총 정의 요소 약 544행. 각 요소는 요소ID·국문명·QName·물리 필드명(snake)·정의·형식·필수(M/C/O)·근거로 형식화, DCAT 3.0 매핑 포함
- 자연재난 L2 5종(기상 T10100·수문 T10200·기후 T10300·지질 T10400·천문 T10500), L3 21종. 분류마다 genus–differentia 판별기준 명시

### 1.2 시범서비스 반영 포인트
- **사건 중심 결합**: 상황 입력 = EVT 골격 생성(주위해 T코드, 행정구역코드, 발생일시, 규모등급) → 모든 검색 결과(레코드)는 REC-02 사건참조로 사건에 종속되는 구조를 API 스키마에 축약 반영
- **레코드 메타**: 청크에 제목·출처기관·문서유형·페이지·LINK 등 정보자원 요소를 태깅 → 근거 인용(F6)의 데이터 기반
- **지역 계층**: 지자체 선택기 = ARU(행정구역코드·지역명·계층경로) 축약판
- **전 행 근거 부착 원칙** → 서비스 응답에도 출처 표기를 필수 규칙으로 채택

## 2. 재난판단기준 5종 v0.6

검토 대상: 태풍(T10105)·풍수해(T10106)·호우(T10107)·홍수(T10206)·산사태(T10401). 시트: Q1 판단조건 / 실제판단데이터 / Q2 등급세분화 / 근거·출처(S1~S15, 2025~2026 웹검증).

### 2.1 판단조건(Q1) — 화면 "판단기준 탭"의 원천
| 유형 | 판단주체 | 정량 기준 |
|---|---|---|
| 태풍 | 기상청 | 정의: 중심 최대풍속 ≥17m/s. 특보: 강풍(경보 ≥21m/s)·풍랑·호우·폭풍해일 기준 |
| 풍수해 | 행안부 | 단일 기준 없음(우산개념) — 구성재해 특보 발효 또는 침수·범람·유실·붕괴 피해 시 |
| 호우 | 기상청 | 주의보 3h 60/12h 110mm · 경보 3h 90/12h 180mm · 극한호우 문자 1h 50&3h 90(또는 1h 72) |
| 홍수 | 환경부 홍수통제소 | 주의보=계획홍수량 50% 유량 시 수위 · 경보=70% (특보지점 223곳) |
| 산사태 | 산림청 | 토양함수지수 주의보 80% / 예비경보 90% / 경보 100% |

### 2.2 등급 세분화(Q2)
- 태풍 강도 5단계(약 17~25 / 중 25~33 / 강 33~44 / 매우강 44~54 / 초강력 ≥54 m/s, 영향 서술 포함)
- 호우·홍수·산사태 특보 2~3단계, 기상 영향예보 4단계
- 홍수 기준수위 단계: 주의보수위→경보수위→계획홍수위→위험홍수위
- 산사태 위험지도 1~5등급 / 자연재해위험개선지구 가·나·다등급
- **공통: 재난 위기경보 4단계** 관심(Blue)·주의(Yellow)·경계(Orange)·심각(Red) — 재난안전법 제38조

### 2.3 실제판단데이터 시트의 시사점
판단에 쓰이는 관측·시스템(AWS/ASOS, 홍수통제소 수위, WAMIS, 산사태예측모델, NDMS 테이블까지)이 정리되어 있으나 실시간 연계는 1차 POC 범위 외. **"계획홍수량·기준수위 산정은 하천기본계획이 원천"**(S3·S4)임이 명시되어 있어, 하천기본계획 코퍼스와 홍수 판단기준의 연결 고리가 됨.

## 3. 자연재해저감종합계획 — 구미시 (2021.9, 제2차)

- 자연재해대책법 제16조 법정계획, 목표 2031년, 구미시 전역 615.30㎢, 대상재해 8유형(해안 제외). 본보고서 약 1,650p + 관리대장 + 별권부록. PDF 텍스트 레이어 정상(추출 가능, 표는 구조 복원 필요).
- **선정 깔때기**: 예비후보지 2,684 → 후보지 345 → 위험지구 대상 148 → **위험지구 66개소 + 관리지구 82개소**. 유형별: 하천22·내수13·토사13·사면10·기타4·가뭄2·바람1·대설1
- **위험도지수**: 간략 A×(0.6B+0.4C)×D, 상세 A×(0.6B+0.4C) (A피해이력·B재해위험도·C주민불편도·D정비여부). 등급: 초고위험(>10)·고위험(5~10)·중위험(3~5)·저위험(1~3)
- **관리대장 = 지구당 1페이지 정형 카드(66건)**: 지구명/유형/위치/좌표/하천명·측점/토지이용·인구·건물/예상피해액/선정사유/위험요인/저감대책(공종·규모)/사업비/투자우선순위(n위/65)/연차별 계획/재원(국·도·시비) — **구조화 DB의 마스터 소스**
- 총사업비 421,604백만원(구조적 65건 413,193 + 비구조적 85건 8,211 등). 투자우선순위 = 기본평가 100점(B/C 15·피해이력 25·재해위험도 30·주민불편 20·경과년수 10) + 부가평가
- 방재성능목표: 1h 60 / 2h 75 / 3h 90mm
- 대표 데이터: 한천지구(하천, 예상피해액 165,509백만원 1위, 사업비 56,050백만원), 구포3지구(내수, 펌프장 Q=1,500㎥/min 신설), 성수천지구(하천, 여유고 1.10m 부족), 황산지구(사면, 낙석방지망 2,500㎡), 비산배수펌프장지구(내수, 증설형)

- 목표 2031년, 의왕시 전역 54.0㎢(6개 행정동·11개 법정동), 8유형. PDF 텍스트 추출은 PyMuPDF 계열 필요(poppler는 한글 유실).
- **선정 깔때기**: 대상 569 → 예비후보지 341 → 후보지 65 → 대상 39 → **위험지구 17개소 + 관리지구 22개소**. 유형별: 하천9·내수3·토사3·대설2 (사면·바람·가뭄·기타 0)
- 위험지구 17개 전체 식별됨(A001 안양천~G002 학현) — 지구코드·위치·사업비까지 확보. 총사업비 107,448백만원(하천 83,468 위주)

- 대표 데이터: 안양천지구(고위험 6.4, 사업비 26,513백만원, 우선순위 1위), 오전천지구(초고위험 12.0, 31,196백만원), 고천A지구(내수, 인구 노출 최대 2,980명), 능안천지구(토사 — 동명의 하천재해 지구와 구분 필요), 청계지구(대설, 자동염수분사 L=2,200m)

## 5. 하천기본계획 — 요천(국가, 2021) · 구미천(지방, 2025)

- 하천법 제25조 법정계획. 표준 장 구성: 1장 개요(하천 제원) → 2장 유역·하천 현황 → **3장 종합분석(계획홍수량→계획홍수위→하폭→시설 능력검토)** → 4장 정비·관리계획(개수지구 카드) → 5장 투자계획(고시용 총괄 요약표)
- **요천**: 국가하천 17.83km(남원시), 유역 485.70㎢, 계획빈도 100~200년, 603p 단일 PDF(물리 페이지↔장-쪽 오프셋 매핑 필요). 서술형+대형 표
- **구미천**: 지방하천 10.07km(구미시), 유역 67.35㎢, 빈도 80~100년, 장별 분권 232p, 개조식 표 중심
- **핵심 표(측점 No.가 조인 키)**: 계획홍수량 결정표(산정지점 Y0~Y5/GM00~GM09) · 빈도별 홍수위표(요천 — 유량-수위 관계 보간 가능) · 계획홍수위표(측점별 홍수위·하폭·좌우안 제방고) · 기성제 능력검토표(과부족 여유고, 음수=월류위험)
- **홍수 판단 연결**: 주의보/경보 기준유량 = 계획홍수량 × 50%/70%. 예: 요천 남원수위표(Y4, No.15+290) 계획홍수량 2,005㎥/s → 주의보 1,003 / 경보 1,404㎥/s. 구미천 종점(GM00) 390㎥/s → 195/273㎥/s. 단, 구미천 유역 내 실측 수위관측소 없음(한계 명시 필요)
- 대표 데이터: 요천 고시 요약표(춘향교 No.14+583 EL.97.39m 등), 보축좌1지구(도통교 좌안 L=650m, 여유고 -1.2~0.2m 미달 — 요천 유일 정비지구), 구미천 No.2+600~2+800 좌안 과부족 -1.11m(월류위험)·배수시설 재가설 34개소·교량 13개소

## 6. 파이프라인 설계에 반영할 공통 발견

1. **추출 도구**: PDF 텍스트는 PyMuPDF(fitz) 사용(poppler 계열은 한글 유실 사례). 표는 pdfplumber/camelot로 구조화 추출. 이 환경의 Read 도구 PDF 렌더링이 아닌 프로그램적 추출로 파이프라인 구성
2. **"1지구=1청크" 자연 단위**: 관리대장(구미 66·의왕 17)과 개수지구 카드가 이미 카드 구조 — 최우선 인덱싱 + districts.json 정형화 소스
3. **하이브리드 필수**: 수치 질의는 정형 JSON(districts/rivers/criteria)에서, 서술 질의는 청크 검색에서 — 대형 측점표(수백 행)는 임베딩 가치가 낮아 요약문만 청킹하고 원자료는 구조화 스토어로
4. **출처 체계**: 전 문서가 "장-쪽번호"(예: 3-403) + 표 번호(<표 3.1-59>) 체계 — 인용 메타데이터로 그대로 사용. 요천은 물리 페이지 오프셋 매핑 테이블 필요
5. **동명 지구·하천 구분**: 지구명+재해유형 복합키(예: 능안천 하천재해 A-07 vs 토사재해 C-01), 추출 시 자간 분리("구 미 시") 정규화 사전 필요
6. **인덱싱 우선순위**: 관리대장 → 제5장(요약) → 제3장 종합부·요약표 → 제4장(사업비·우선순위) → 부록 분석표(제3장과 중복 주의) → 제2장(시설 목록 표만 선별)
7. **HWP 한계**: 남원 저감계획·안양천(의왕) 하천기본계획은 .hwp만 존재 — 1차 제외, 고도화 시 hwp 변환 공정(hwp5txt 등) 또는 PDF 원본 확보
