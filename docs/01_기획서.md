## 1. 추진 배경 및 목적

### 1.1 배경
- 부처·기관별로 분절된 재난안전 데이터(관측, 특보, 계획 문서, 피해 집계)를 개별 사건 단위로 결합·활용하려는 「재난안전 AI데이터 구축」 사업(NIA, TTAK.KO-10.1126/1047 기반)이 진행 중이다.
- 지자체 재난 대응의 핵심 참조 문서인 **자연재해저감종합계획**과 **하천기본계획**은 수백~수천 페이지의 PDF/HWP 보고서로 존재하여, 상황 발생 시 필요한 정보(위험지구, 계획홍수량, 저감대책 등)를 즉시 찾기 어렵다.
- T3Q에서 재난메타 정의서·재난판단기준·데이터 정리 문서를 설계 중이며, 이 메타 체계를 기준으로 한 검색·시각화 서비스의 실증(POC)이 필요하다.

### 1.2 목적
재난 상황 입력 → 관련 계획 문서 검색(RAG) → LLM 질의응답 → **재난메타+데이터메타 기준의 시각화 표출**까지의 전 과정을 시범 구현하여,
1. 계획 문서(비정형 PDF)를 재난메타 체계로 구조화·검색 가능함을 실증하고,
2. 판단기준(특보·등급)과 결합한 상황 표출 화면의 UX 방향을 제시한다.

## 3. 사용자 및 시나리오

### 3.1 대상 사용자
- **1차**: 지자체·중앙부처 재난상황실 담당자 (상황 판단·보고 지원)
- **2차**: 방재 계획 담당 공무원, 유관기관 분석가

### 3.2 핵심 사용자 시나리오
**S1. 호우 상황 — 위험지구 조회 (의왕)**
> 의왕시에 호우경보(3h 90mm 초과 예상) 발효. 담당자가 재난상황 패널에 [호우 / 의왕시 / 경보]를 입력하고 "침수 위험이 큰 지구와 저감대책은?"을 질의 → 저감종합계획 내 내수재해·하천재해 위험지구 목록, 지구별 위험요인·저감대책·사업비가 카드/표로 표출되고, 근거 페이지가 인용된다.

**S2. 홍수 상황 — 하천 기준 조회 (남원/요천)**
> 요천 수위 상승. [홍수 / 남원시 / 주의보]를 입력하고 "요천의 계획홍수량과 주의보 기준은?"을 질의 → 하천기본계획의 지점별 계획홍수량 표와, 판단기준(계획홍수량 50%/70% = 주의보/경보 수위 개념)이 함께 표출된다.

**S3. 산사태 상황 — 사면재해 대책 조회 (구미)**
> 토양함수지수 90%(예비경보) 도달. [산사태 / 구미시 / 예비경보] 입력, "사면재해 위험지구 현황과 등급별 대응은?" 질의 → 저감종합계획의 사면재해 위험지구, 등급(가/나/다), 판단기준 3단계(80/90/100%)가 게이지·배지로 표출된다.

## 4. 핵심 기능 (1차 POC)

| # | 기능 | 설명 |
|---|---|---|
| F1 | 재난상황 입력 패널 | 재난메타 기준 입력: 재난유형(T코드 5종)·지자체(3개)·경보단계(특보/위기경보 4단계)·상황 텍스트. 재난메타의 사건(EVT) 골격(유형·지역·시점·등급)에 대응 |
| F2 | 문서 검색 (RAG Retrieval) | 저감계획·하천기본계획 코퍼스에서 상황 조건(유형·지자체 필터)+질의로 관련 청크 검색 |
| F3 | AI 질의응답 | 검색 청크를 근거로 LLM이 응답 생성. 근거 문서·페이지 인용 필수. API 미설정 시 mock 응답 폴백 |
| F4 | 판단기준 시각화 | 선택된 재난유형의 판단조건·등급 세분화(판단기준_5종)를 기준표·게이지·단계 배지로 표출 |
| F5 | 구조화 데이터 표출 | 응답과 연계된 정형 데이터 카드: 위험지구(지구명·유형·등급·저감대책·사업비), 하천 제원(연장·유역면적·계획홍수량) |
| F6 | 근거 추적 | 모든 응답에 출처(문서명·장·페이지) 표기 — 메타 정의서의 근거 부착 원칙 준수 |

### 제외 (1차 POC 범위 외)
- 실시간 관측 연계(AWS·수위관측 API), 지도(GIS) 표출, 사용자 인증, HWP 파싱, 부산/인제/영천 확장, 상황일지·중대본보고서 코퍼스

## 5. 화면 표출 데이터 유형 (재난메타 → UI 매핑)

재난메타 v0.6 분석 결과, 시범서비스의 검색/표출 데이터는 다음 유형으로 정리된다.

| 메타 구분 | 대표 요소 | UI 표출 형태 |
|---|---|---|
| 사건(Event) 골격 | EVT-01 사건ID, EVT-02 사건명, EVT-04 발생일시, EVT-08 주위해(T), EVT-10 규모등급, 위기경보단계 | 상황 헤더 바 + 단계 배지(관심/주의/경계/심각 = Blue/Yellow/Orange/Red) |
| 판단조건 (Q1) | 유형별 정량 기준(mm·m/s·%·수위) | 기준표 + 현황 대비 게이지 |
| 등급 세분화 (Q2) | 특보 2~3단계, 태풍 강도 5단계, 위험지도 1~5등급, 위험개선지구 가/나/다 | 단계 스테퍼·배지 |
| 정보자원(레코드) | 제목·출처기관·문서유형·LINK·페이지 | 근거 인용 카드 |
| 지역(AreaUnit) | 행정구역코드·지역명·계층경로 | 지자체 선택기 + 결과 필터 |
| 문서 구조화 데이터 | 위험지구 속성, 하천 제원·계획홍수량 | 데이터 테이블·상세 카드 |

## 6. 성공 기준 (1차 POC)
1. 3개 시나리오(S1~S3)가 데모 가능: 입력→검색→응답→시각화 전 과정 동작
2. 코퍼스: PDF 4종(구미·의왕 저감계획, 요천·구미천 하천기본계획) 청킹·메타 태깅 완료
3. 모든 AI 응답에 근거 문서·페이지 인용 표기
4. UI가 UNE Design System 토큰·컴포넌트와 SpoqaHanSansNeo 글꼴만으로 구성
5. 판단기준 5종이 화면에서 조회 가능

## 7. 추진 일정 (Phase — 상세는 CLAUDE.md)
1. **Phase 1** 스캐폴딩·데이터 파이프라인 (PDF 추출→청킹→메타 태깅→코퍼스)
2. **Phase 2** 검색·RAG API (인덱스, FastAPI, LLM 연동+mock)
3. **Phase 3** 프론트엔드 UI (UNE DS 3패널 화면)
4. **Phase 4** 통합·시나리오 검증·문서화
